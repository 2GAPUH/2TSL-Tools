# 2TSL-Tools — Полное описание проекта

## Обзор

**Название:** 2TSL toolbox  
**Версия:** 0.5.2  
**Тип:** Расширение для Google Chrome (Manifest V3)  
**Язык:** Русский (весь интерфейс и комментарии — на русском)  
**Автор:** 2GAPUH  
**Репозиторий:** https://github.com/2GAPUH/2TSL-Tools  

**Назначение:** Набор инструментов для повышения эффективности работы сотрудников технической поддержки Ростелеком — управление шаблонами ответов, навигация между внутренними системами, учёт заявок, система напоминаний и автоматизация заполнения форм.

**Технологический стек:** Чистый (vanilla) JavaScript без внешних зависимостей. Никаких npm/yarn, никаких фреймворков, никаких сборщиков. Весь проект — самодостаточный набор JS-файлов, работающих через Chrome Extension API.

## Архитектура расширения

Расширение построено по классической архитектуре Chrome Extension с тремя типами скриптов:

### 1. Popup (Всплывающее окно)
- **popup.html** — основной интерфейс пользователя с 4 вкладками
- **popup.js** — бизнес-логика интерфейса

### 2. Background (Фоновый service worker)
- **background.js** — обработка регистрации пользователей, система напоминаний (alarms/notifications), маршрутизация сообщений между вкладками

### 3. Content Scripts (Внедряемые скрипты)
- 9 content-скриптов, каждый привязан к определённой внутренней системе Ростелеком через manifest.json

### popup.html / popup.js — Основной интерфейс

#### Вкладка «Шаблоны»

Полноценный CRUD-менеджер текстовых шаблонов:

- **Создание/редактирование/удаление** шаблонов с заголовком и текстом
- **Группировка** шаблонов по произвольным категориям
- **Поиск** по названию и содержимому
- **Фильтрация** по группам
- **Копирование** шаблона в буфер обмена
- **Вставка** шаблона прямо в чат Omnichat (через отправку сообщения в content script)
- **Управление группами:** создание, переименование, удаление (с переносом шаблонов)

#### Вкладка «Учёт заявок»

Система ежедневного учёта рабочего процесса:

- **Добавление записей** двух типов: «Закрытые заявки» и «Выезды»
- **Каждая запись** содержит: номер заявки, комментарий, часы и минуты
- **Статистика в реальном времени:**
  - Количество закрытых заявок
  - Количество выездов
  - Общее количество
  - Производительность (заявки в час за вычетом обеденного перерыва)
  - Процент закрытия
- **Хронологическая лента** всех записей
- **Управление рабочим днём:** начало нового дня, завершение дня с экспортом
- **CSV-экспорт** с BOM-маркером для корректного отображения в русском Excel
- **Синхронизация** с боковой панелью в Omnichat через `chrome.storage`

#### Вкладка «Напоминалка»

Просмотр и управление напоминаниями:

- **Список** всех созданных напоминаний с описанием и временем
- **Редактирование** времени напоминания
- **Удаление** напоминаний
- Напоминания создаются из TTM (content-ttm.js) и обрабатываются background.js

#### Вкладка «Настройки»

Центр управления расширением:

- **Переключатели** для всех функций (9 переключателей)
- **Тёмная тема** для интерфейса popup
- **Счётчик пользователей** — отображение количества активных пользователей
- **Информация о данных** — объём сохранённых данных
- **Очистка данных** — удаление всех сохранённых данных
- **Форма обратной связи** (HTML-интерфейс существует, отправка не реализована)

#### Ключевые функции popup.js

| Функция | Назначение |
|---------|------------|
| `loadAllData()` | Загрузка всех данных из chrome.storage при инициализации |
| `renderTemplates()` | Рендеринг списка шаблонов |
| `renderGroupFilter()` | Рендеринг фильтра по группам |
| `copyTemplateToClipboard(id)` | Копирование шаблона в буфер обмена |
| `pasteTemplateToMessage(id)` | Вставка шаблона в чат через chrome.tabs.sendMessage |
| `updateTicketsUI(date, data)` | Рендеринг учёта заявок со статистикой |
| `addTicketEntry(type)` | Добавление записи о заявке |
| `startNewDay()` / `finishDay()` | Управление рабочим днём + CSV-экспорт |
| `renderReminders()` | Рендеринг списка напоминаний |
| `applySettings()` / `applyDarkMode()` | Применение настроек |
| `resizePopup()` | Динамический пересчёт высоты popup |

### background.js — Фоновый service worker

Основные задачи:

#### Регистрация пользователей
- Генерация уникального `userId` (сохраняется в chrome.storage)
- Автоматическая POST-регистрация на внешний API: `http://158.160.132.18:3001/api/user/register`
- Передача userId + версии расширения

#### Система напоминаний
- `addReminder(data)` — создание напоминания, сохранение в chrome.storage, создание chrome.alarms
- `removeReminder(reminderId)` — удаление напоминания + очистка alarm
- `updateReminder(data)` — обновление времени/описания, пересоздание alarm
- `getReminders()` — получение всех напоминаний
- `cleanupOldReminders()` — периодическая очистка (каждые 5 минут) напоминаний, показанных более 30 минут назад

#### Уведомления
- `chrome.notifications.create()` для напоминаний
- Клик по уведомлению открывает URL заявки

#### Восстановление при запуске
- При `chrome.runtime.onInstalled` и `chrome.runtime.onStartup`: восстановление alarms для всех pending-напоминаний
- Немедленный показ просроченных напоминаний

#### Обработчики сообщений
| Сообщение | Действие |
|-----------|----------|
| `openForm` | Открытие новой вкладки с формой ассистента |
| `addReminder` | Создание нового напоминания |
| `removeReminder` | Удаление напоминания |
| `updateReminder` | Обновление напоминания |
| `getReminders` | Получение списка напоминаний |

### content-omnichat.js — Интеграция с Omnichat

**Целевая система:** `omnichat.rt.ru`  
**Версия модуля:** 8.0

#### Функциональность 1: Вкладка пользовательских шаблонов

Внедряет третью вкладку «Дополнительно» в модальное окно шаблонов Omnichat:

- **Кнопка** добавляется рядом с системными вкладками
- **Список** пользовательских шаблонов из chrome.storage
- **Поиск** по названию и содержимому
- **Фильтрация** по группам
- **Вставка** текста в редактор Draft.js Omnichat через ClipboardEvent

#### Функциональность 2: Кликабельные ссылки на НЛС и заявки

Автоматическое сканирование сообщений чата и преобразование номеров в кликабельные ссылки:

- **НЛС (номер лицевого счёта):** 12-значные числа, начинающиеся на 2, 3 или 4  
  Регулярное выражение: `\b(\d{3}\s?\d{3}\s?\d{3}\s?\d{3})\b`
- **Номера заявок:** определяются по ключевым словам «задание», «заявка», «номер задания» + 7-10 цифр
- **Обращения:** по ключевому слову «обращение №» + 7-10 цифр
- Клик по ссылке открывает TTM с поисковым запросом
- **MutationObserver** отслеживает новые сообщения для их обработки

#### Ключевые функции

| Функция | Назначение |
|---------|------------|
| `addAdditionalButton()` | Внедрение кнопки вкладки «Дополнительно» |
| `loadAndDisplayCustomTemplates()` | Рендеринг пользовательских шаблонов |
| `insertTextIntoDraftEditor()` | Вставка текста в редактор Draft.js |
| `processTextNode()` | Поиск и преобразование НЛС/заявок в ссылки |
| `processChatMessages()` | Сканирование контейнеров сообщений |
| `openInTTM()` | Открытие TTM с поисковым значением |
| `detectTheme()` | Определение тёмной/светлой темы Omnichat |

### content-ttm.js — Интеграция с TTM

**Целевая система:** `www.ttm.rt.ru`  
**Версия модуля:** 1.8

#### Описание

Набор утилитарных кнопок на панели инструментов TTM (Trouble Ticket Management):

#### Кнопки

1. **Кнопка «Ассистент»** (иконка support_agent)  
   - Извлекает данные заявки: номер инцидента, название услуги, РФ клиента
   - Открывает форму использования ассистента с предзаполненными полями

2. **Кнопка «Таймер»** (иконка timer)  
   - Создаёт напоминание через модальное окно
   - Два режима: обратный отсчёт (минуты) или указание точного времени

3. **Кнопка «Onyma»**  
   - Открывает поиск Onyma с ИЛС клиента (номер начинается с «4»)
   - Отображается только при наличии подходящего ИЛС

4. **Кнопка «SIPAL»**  
   - Открывает поиск SIPAL с ИЛС клиента (номер начинается с «2»)
   - Отображается только при наличии подходящего ИЛС

5. **Автопоиск**  
   - Автоматическое заполнение поля поиска TTM при переходе из Omnichat

#### Извлечение данных

| Данные | Метод извлечения |
|--------|-----------------|
| Номер инцидента | Из нескольких DOM-элементов |
| Название услуги | Из заголовка заявки |
| Технология | Парсинг из названия услуги (FTTx, xDSL, xPON, DOCSIS) |
| РФ подключения | Из данных клиента |
| ИЛС | Из данных лицевого счёта |

#### SPA-навигация

TTM — одностраничное приложение. Для отслеживания навигации используются:
- `setInterval` с проверкой URL
- Патчинг `history.pushState` / `history.replaceState`
- События `popstate` и `hashchange`

### content-grafana.js — Grafana → SSH переход

**Целевая система:** `epd.rt.ru/stb-events/*`

#### Описание

Делает IP-адреса в дашбордах Grafana кликабельными для перехода к SSH-терминалам:

- Сканирует ячейки таблиц Grafana `[role="cell"]` на наличие IP-адресов
- Формат IP: `внутренний / внешний` (извлекается внешний)
- Определяет регион по тексту ячейки или URL-параметру `var-vDataSource`

#### Маппинг регионов → SSH-серверы

| Регион | SSH-сервер |
|--------|------------|
| Волга | `http://10.77.19.50/stb_ssh/` |
| Юг | `http://10.144.39.131/stb_ssh/` |
| Северо-Запад | `http://10.160.191.178/stb_ssh/` |

При клике на IP: сохранение в chrome.storage → открытие SSH-страницы в новой вкладке.  
MutationObserver отслеживает динамически загружаемые данные Grafana.

**Целевые системы:** `10.77.19.50/stb_ssh/*`, `10.144.39.131/stb_ssh/*`, `10.160.191.178/stb_ssh/*`

Автоматическое заполнение IP-адреса на SSH-терминале:

- Заполняет `input[name="ip"]` и устанавливает фокус

| `savedFormData` | Object | content-form.js | popup.js | Сохранённые регион/ФИО |

## Резюме

**2TSL-Tools** — это зрелое расширение для Chrome, созданное для оптимизации работы инженеров технической поддержки Ростелеком. Оно решает ключевые проблемы ежедневного рабочего процесса:

1. **Управление шаблонами** — создание и использование текстовых шаблонов прямо в интерфейсе Omnichat
2. **Навигация между системами** — мгновенный переход между 9 внутренними системами с автоматическим заполнением данных
3. **Учёт заявок** — отслеживание производительности и экспорт статистики
4. **Напоминания** — система таймеров для отслеживания времени ожидания
5. **Автоматизация** — устранение рутинного копирования/вставки данных между системами

Расширение полностью самодостаточно (0 внешних зависимостей), использует современный Manifest V3 и покрывает весь рабочий процесс от чтения чата до заполнения форм и анализа grafana-дашбордов.
